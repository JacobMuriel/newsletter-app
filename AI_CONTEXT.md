# AI Context

## Purpose

Batch pipeline that builds a sectioned daily news newsletter from RSS feeds, enriches selected stories with OpenAI or fallback summaries, writes Markdown output, and can optionally send the result by SMTP.

## Current State

- Small Python codebase with one entry script: [main.py](/Users/jacobmuriel/Desktop/newsletter/main.py)
- Config-driven via [config/settings.yaml](/Users/jacobmuriel/Desktop/newsletter/config/settings.yaml) and [config/sources.yaml](/Users/jacobmuriel/Desktop/newsletter/config/sources.yaml)
- No tests or lint config found
- Markdown output is the real supported format
- HTML artifact in `output/` exists but is not generated by current source. Needs verification.

## Main Commands

```bash
pip install -r requirements.txt
DRY_RUN=true SEND_EMAIL=false OPENAI_ENABLED=false python main.py
OPENAI_ENABLED=true DRY_RUN=true SEND_EMAIL=false python main.py
```

## Architecture Summary

Flow is fixed and linear: load config -> fetch RSS -> dedupe -> categorize -> quality filter -> rank -> select by section -> summarize selected stories -> write Markdown -> optionally email.

Sections are hardcoded in code as well as limited in config:

- `top`
- `markets`
- `ai`
- `finance_market_structure`
- `nba`

## Known Pitfalls

- `.venv` is machine-specific and currently points outside the repo.
- Categorization is substring-based and can misclassify broad terms like `training`, `ice`, or `treasury`.
- Deduplication is heuristic and pairwise.
- Several feeds emit malformed-feed warnings in real runs.
- `temperature` exists in summarization config but is not used by the current OpenAI request.
- Workflow file exists, but this checkout is not a Git repo, so automation status is unknown.

## Code Conventions Used Here

- Dataclass-based shared models
- Small single-purpose pipeline modules
- Config-first tuning in YAML instead of hardcoding most thresholds
- Simple logging via the stdlib `logging` module
- Minimal abstractions; direct function calls from `main.py`

## Do Not Change Casually

- Section names and ordering: used in selection and rendering
- Environment variable names in [.env.example](/Users/jacobmuriel/Desktop/newsletter/.env.example) and [main.py](/Users/jacobmuriel/Desktop/newsletter/main.py)
- Ranking and categorization weights without checking downstream output quality
- SMTP behavior in [news_pipeline/send_email.py](/Users/jacobmuriel/Desktop/newsletter/news_pipeline/send_email.py) if production delivery depends on it
